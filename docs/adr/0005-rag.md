# ADR-0005: RAG (Retrieval-Augmented Generation)

## Status

Proposed

## Context

Para melhorar a qualidade das respostas da IA, o sistema precisa acessar conhecimento externo (documentação, código, bases de conhecimento). As opções são:

- **Contexto manual**: usuário cola todo o contexto no prompt
- **RAG simples**: embedding + busca por similaridade + injeção no prompt
- **RAG avançado**: chunking, reranking, query expansion, hibrido (vector + keyword)

## Decisão

Implementar RAG com pipeline completo: chunking → embedding → search → rerank → generation.

## Justificativa

1. **Qualidade**: RAG produz respostas mais precisas e factuais que geração pura
2. **Contexto ilimitado**: não limitado ao context window do modelo
3. **Atualizável**: base de conhecimento pode ser atualizada sem retreinar
4. **Transparência**: respostas podem citar fontes consultadas
5. **Offline-first**: todo o pipeline roda localmente (embedding + search)

## Consequências

### Positivas
- Respostas mais precisas e com fontes
- Suporte a bases de conhecimento grandes
- Privacidade (dados nunca saem da máquina)
- Base de conhecimento versionável em git

### Negativas
- Complexidade adicional (chunking, embedding, search, rerank)
- Latência maior que geração pura
- Qualidade depende da estratégia de chunking
- Necessidade de storage vetorial (SQLite com vetores)

## Componentes do Pipeline RAG

1. **Ingestion**: chunking de documentos → embedding → store
2. **Retrieval**: embedding da query → busca por similaridade → rerank
3. **Generation**: contexto recuperado + prompt original → geração

## Stack

- Embedding: BGE Small EN v1.5 (local, OpenVINO)
- Storage: SQLite com vetores (extensão sqlite-vec ou similar)
- Chunking: recursive character text splitter
- Rerank: BGE Reranker v2 (local, OpenVINO)
